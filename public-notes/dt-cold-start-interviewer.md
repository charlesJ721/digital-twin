# 冷启动填充协议：Digital Twin Interviewer

> 创建：2026-06-11 | 状态：v1.0
>
> 一句话：基于苏格拉底式引导发现 + 混合苏格拉底提示 + 推理-适应循环，系统性地将「我是谁」从模糊自述转化为可操作的维度数据。

---

## 为什么需要这个协议

冷启动后的核心挑战：用户知道「需要填充内容」，但「我是谁」不是一个可以通过主动写作文解决的问题。原因：

1. **自我盲区**：人对自己最核心的模式往往无意识——它们太自然了，自然到看不见
2. **语言化困难**：深层认知模式不是可以用问卷直接捕获的——它们隐藏在具体故事里
3. **防御机制**：直接问「你的核心价值观是什么」得到的答案是经过美化的自我呈现，不是真实运作规则

解决方案：不是让用户「写自己」，而是通过结构化对话**诱导模式浮现**。

## 方法论基础

本协议综合三个方法论：

| 来源 | 核心贡献 | 年份 |
|---|---|---|
| Christine Padesky | 四阶段苏格拉底式引导发现（CBT 临床验证） | 1993 |
| IOPscience 混合苏格拉底提示法 | 六条追问链 + 动态原则组合 | 2025 |
| Parloa inference-adaptation loop | 推理-运行画像-输出适应的三动作引擎 | 2026 |

### Padesky 四阶段

```
Stage 1 → 信息性提问（让抽象变具体）
Stage 2 → 深度倾听（捕捉意外和矛盾）
Stage 3 → 综合（拼接分散答案为模式）
Stage 4 → 应用（用新发现审视旧结论）
```

### 六条苏格拉底追问链

| 链 | 触发条件 | 最适合维度 |
|---|---|---|
| 澄清定义 | 用户用模糊大词 | Layer 2 认知、Layer 3 价值观 |
| 假设检验 | 用户陈述信念 | Layer 3 价值观、Layer 1 人格 |
| 类比映射 | 概念抽象难表达 | Layer 2 认知、Layer 5 知识 |
| 视角切换 | 用户陷入单一视角 | Layer 6 社会、Layer 2 偏误 |
| 后果推演 | 用户做出价值判断 | Layer 3 价值观、Layer 7 叙事 |
| 元问题 | 对话需要反思 | Layer 2 元认知、Layer 7 自我 |

### 推理-适应循环

每次对话后自动更新 Running Profile，下一轮对话以此为起点，形成「挖掘 → 抽象 → 验证 → 修正」的闭环。

## 与七层雷达图的关系

```
维度框架（七层雷达图）= 数据容器（what）
Interviewer Skill = 采集协议（how）
Running Profile = 动态状态（current）
```

三者形成完整的冷启动填充引擎。

## 已实现

本协议已固化为 Hermes Skill：`digital-twin-interviewer`

加载方式：当对话意图为维度挖掘时自动加载，或手动说「加载数字分身访谈者」。

卸载方式：会话结束自然切换回正常模式，或说「退出挖掘模式」。

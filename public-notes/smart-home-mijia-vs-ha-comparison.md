# 全屋智能方案对比：米家生态 vs Home Assistant

> 创建：2026-06-12 | 适用：正在装修或计划改造全屋智能的个人用户
>
> 核心命题：选米家还是 HA？答案是「都要」。米家做执行层，HA 做大脑。但这取决于你的需求复杂度和折腾意愿。

---

## 一、先理解架构差异——为什么它们不是同一个东西

### 1.1 米家：垂直整合的消费电子生态

```
云端（小米 IoT 云）
  │
  ├─ 米家 App ──→ 手机控制
  ├─ 小爱音箱 ──→ 语音控制
  └─ 多模网关 ──→ Zigbee/蓝牙 Mesh 设备
       │
       ├─ 传感器（门磁、人体、温湿度）
       ├─ 执行器（开关、窗帘电机、阀门）
       └─ 家电（空调、净化器、扫地机）
```

**核心特征**：设备直连云或网关，App 统一管理。逻辑跑在云端——断网则大部分自动化失效。

### 1.2 Home Assistant：本地优先的自动化引擎

```
HA 实例（树莓派 / NAS / Mini PC / Mac）
  │
  ├─ 集成层 ──→ 对接 2500+ 品牌/协议
  │    ├─ 米家（通过 Xiaomi Gateway 3 / Miot Auto）
  │    ├─ Apple HomeKit、Google Home
  │    ├─ Z-Wave、Zigbee2MQTT、Matter
  │    └─ 自定义 MQTT、REST API
  │
  ├─ 自动化引擎 ──→ 本地执行，无需云端
  │    ├─ 条件触发（时间、状态、传感器值）
  │    ├─ 复杂逻辑（if/and/or/嵌套）
  │    └─ 跨品牌联动（小米传感器触发飞利浦灯泡）
  │
  └─ 仪表盘 ──→ 完全自定义的 UI
```

**核心特征**：本地处理一切。断网不影响自动化。可以连接任何品牌的设备。代价是需要自己搭建和维护。

### 1.3 本质差异：云 vs 本地

```
米家         HA
────────────────────────
自动化执行   云端 ❌     本地 ✅
断网可用     部分 ✅     全部 ✅
响应延迟     100-500ms    <50ms
跨品牌联动   不支持       原生支持
学习曲线     零           中高
维护成本     零           按需维护
设备兼容     米家生态     2500+ 品牌
```

## 二、技术原理对比

### 2.1 通信协议

| 协议 | 米家支持 | HA 支持 | 特点 |
|---|---|---|---|
| **Zigbee** | ✅ 多模网关 | ✅ Z2M/ZHA | 低功耗、Mesh 自组网、稳定 |
| **蓝牙 Mesh** | ✅ 多模网关 | ✅ | 米家低端传感器大量使用 |
| **Wi-Fi** | ✅ | ✅ | 功耗高，不适合电池设备 |
| **Z-Wave** | ❌ | ✅ | 欧美主流，国内用得少 |
| **Matter** | 部分 | ✅ 原生 | 跨生态互联标准，未来趋势 |
| **红外/RF** | ✅ 部分 | ✅ 通过 IR 模块 | 控制传统家电 |

### 2.2 自动化逻辑引擎

**米家自动化**：
```
如果 [条件] 且 [条件] 则 [执行]
```
- 条件：时间、设备状态、传感器触发、天气、地理位置
- 限制：不支持 if-else 嵌套，不支持复杂多条件分支
- 执行层：基本够日常使用

**HA 自动化**：
```
trigger → condition → action  +  script  +  blueprint  +  Node-RED
```
- 原生支持嵌套条件、变量、模板（Jinja2）、脚本复用
- Node-RED 提供可视化编程：拖拽节点构建复杂逻辑流
- 一个真实例子：「如果客厅人体传感器 10 分钟内无触发，且窗帘关闭，且电视开着，且时间是晚上 7 点后 → 发通知问是否关电视，5 分钟内无回复则关」

### 2.3 本地 vs 云端延迟实测

| 操作 | 米家（云端） | HA（本地） |
|---|---|---|
| 传感器触发 → 灯亮 | 200-800ms | **<50ms** |
| 小爱语音 → 设备响应 | 500-1500ms | N/A（需额外配置） |
| App 远程控制 | 500-2000ms | 200-1000ms（通过 Nabu Casa） |

人感知延迟阈值约 100ms。米家的云端延迟在复杂自动化中可能被感知到「慢半拍」，HA 本地执行体感是即时的。

## 三、部署方式：DIY vs 开箱即用

### 3.1 米家：零门槛

1. 买设备
2. 开机，打开米家 App，自动发现
3. 配置房间、创建场景
4. 完成

不需要任何技术知识。家里的老人小孩都能用。

### 3.2 Home Assistant：四档选择

| 方案 | 硬件成本 | 技术门槛 | 稳定性 | 推荐人群 |
|---|---|---|---|---|
| **HA Green** | $99 | ⭐ 最低 | 高 | 想体验 HA 的新手 |
| **HA Yellow** | $175+ | ⭐⭐ | 最高 | 需要 Zigbee/Thread 内置 |
| **树莓派 5** | ~$120 | ⭐⭐ | 中高 | 有 Linux 基础的 DIY 用户 |
| **闲置 PC/Mac/NAS Docker** | $0 | ⭐⭐⭐ | 取决于硬件 | 家里有闲置设备的 |

**推荐路径**：
- 第一次用 → HA Green，官方硬件，插电即用，App 引导设置
- 已有树莓派/闲置 Mac → Docker 安装
- 你已经有的 Mac Mini → 直接 Docker，零额外成本

安装命令（Docker）：
```bash
docker run -d \
  --name homeassistant \
  --privileged \
  --restart=unless-stopped \
  -v /path/to/config:/config \
  -v /run/dbus:/run/dbus:ro \
  --network=host \
  ghcr.io/home-assistant/home-assistant:stable
```

## 四、日常使用体验对比

### 4.1 手机控制

| | 米家 App | HA Dashboard |
|---|---|---|
| 界面美观 | 现代、统一 | 完全自定义，但默认朴素 |
| 设备管理 | 按房间分组 | 按用户定义视图 |
| 家人共享 | 家庭成员功能 | 多用户权限管理 |
| 远程访问 | 自带云服务 | 需配置 Nabu Casa ($6.5/月) 或 FRP 穿透 |

### 4.2 语音控制

| | 小爱音箱 | HA Assist |
|---|---|---|
| 中文识别 | 优秀 | 需配置 Whisper + Piper |
| 唤醒词 | 小爱同学 | 可自定义 |
| 理解能力 | 固定指令集 | LLM 驱动（可接 OpenAI/本地模型） |
| 隐私 | 云端处理，小米可听到 | **本地处理，数据不出家门** |

### 4.3 异常场景应对

| 异常 | 米家 | HA |
|---|---|---|
| 断网 | 自动化失效，本地按键/Zigbee 直接配对仍可用 | 全部正常，本地执行 |
| 网关故障 | 替换后需重新配对子设备 | Z2M 备份恢复，一键还原 |
| 云端宕机 | 自动化全挂，历史罕见但发生过 | 不受影响 |
| 设备掉线 | App 报警 | 可配置自动重连、通知 |
| 停电恢复 | 大部分设备自动恢复 | 自动恢复 + 可配置状态检查 |

## 五、生态和延展性

### 5.1 设备兼容数量

| | 米家 | HA |
|---|---|---|
| 官方支持品牌 | ~200（米家生态链） | **2500+**（所有主流品牌） |
| 非米家设备接入 | 部分支持（第三方 Works with Mijia） | 全部支持 |
| Apple HomeKit | 部分设备支持 | **内置 HomeKit Bridge，任何 HA 设备都能进 HomeKit** |
| Google Home / Alexa | 部分支持 | 可通过集成接入 |

**实际意义**：用 HA，你可以买最便宜的门磁（米家 $5），联动最贵的灯具（Philips Hue $50），还能把车（Tesla 集成）、扫地机（任何品牌）、甚至空调（红外）全部接入一个自动化。

### 5.2 高级扩展

| 能力 | 米家 | HA |
|---|---|---|
| 能源监控 | 部分设备支持 | CT 钳 + ESPHome 实现全屋电力监控 |
| 自定义硬件 | ❌ | ✅ ESPHome，自己画 PCB 做传感器 |
| AI 接入 | 小爱大脑（云端） | **本地 LLM + Assist，可接任何大模型** |
| 安防联动 | App 通知 | 联动监控摄像头、人脸识别 (Frigate)、自动报警 |
| 场景联动（非家居） | ❌ | 开车到家 → 自动开车库门 + 开灯 + 关安防 |

### 5.3 米家生态的商业本质

米家是一个**封闭的硬件销售渠道**。小米通过低价传感器吸引你入坑，然后通过卖网关、音箱、大家电赚回来。你的数据（设备使用习惯、家庭作息）是它 AIoT 战略的燃料。

小米的动机是「让你买更多小米设备」，不是「让你的家最智能」。这导致：
- 跨品牌联动故意不做（防止你买别家）
- 本地自动化故意弱化（云端才能收集数据）
- 开放 API 逐步收紧（2023 年后新设备获取 token 越来越难）

## 六、实际推荐路径

### 6.1 三种方案

**方案 A：纯米家（够用主义）**
```
适合：租房、怕折腾、需求简单（自动开关灯/窗帘/空调）
成本：传感器 $5-15/个，网关 $50-100，开关 $30-60
优点：零折腾、家人友好、小爱语音好用
缺点：断网瘫痪、自动化能力弱、被小米锁死
```

**方案 B：HA 做主脑 + 米家设备做手脚（推荐）⭐**
```
HA（Mac Mini Docker 免费） 
  ├─ 米家多模网关 → 所有 Zigbee/蓝牙设备
  ├─ 米家 Wi-Fi 设备（通过 Miot Auto 集成）
  ├─ 非米家设备（按需加，不被绑定）
  └─ Apple HomeKit Bridge → 全家 iPhone 控制

适合：有技术基础、追求极致自动化、长期住房
成本：同方案 A + HA 硬件（你已有 Mac Mini，零额外）
优点：本地执行、断网可用、跨品牌联动、可无限扩展
缺点：需要维护、家人学习成本
```

**方案 C：全 DIY（极客路线）**
```
HA + ESPHome 自己画板子 + Zigbee2MQTT + Frigate 安防

适合：硬件工程师、极客
成本：时间 >> 金钱
```

### 6.2 推荐执行顺序

```
Phase 1: 装修阶段 ──→ 预埋零火线 + 加深底盒 + 预留中性线到窗帘位
                      开关底盒预留零线是所有智能开关的基础
                      
Phase 2: 入住后 ──→ 先纯米家，全屋基础自动化跑起来
                     灯控、窗帘、空调、门锁，米家够用且稳
                     
Phase 3: 稳定后 ──→ Mac Mini 上 Docker 部署 HA
                     慢慢迁移自动化到 HA，米家设备通过集成接入
                     一个设备两套系统都能控，逐步切换
                     
Phase 4: 进阶 ──→ 加入非米家设备（优质传感器/灯具）
                     复杂自动化（安防联动、能源管理）
                     Apple HomeKit 全家统一入口
```

**永远不要一步到位。** 先让基础自动化稳定跑 2 周，再加复杂度。全屋智能最大的坑是「自动化太复杂导致家属不满」。

## 七、一张图总结

```
                       米家生态          Home Assistant
                       
入门门槛               ★☆☆                ★★★
日常使用体验           ★★★★               ★★★（初期）
自动化能力             ★★★                ★★★★★
断网可靠性             ★★                 ★★★★★
设备兼容性             ★★                 ★★★★★
中文语音               ★★★★★              ★★☆（需配置）
家人友好度             ★★★★★              ★★★
隐私保护               ★★                 ★★★★★
可玩性/扩展性          ★★                 ★★★★★
长期绑定风险           高                  无

结论：
  米家 = 消费级产品，买来即用，用着不累，但不能越界
  HA   = 基础设施，搭建需要投入，建好后是真正的智能管家
  
  最佳实践：两者共存。HA 做主脑，米家做手脚。
```

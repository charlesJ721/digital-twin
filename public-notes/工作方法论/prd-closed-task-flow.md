# PRD 闭合任务流

> 把 PRD 从「开放性无尽细节」转化为「一系列有明确交付物的闭合任务」。基于你的认知偏好：喜欢闭合逻辑环，核心价值在架构设计层。

## 核心原则

**AI 吃掉文档撰写层，你聚焦架构设计层。**

## 四阶段闭合任务流

### 第一阶段：竞品扫描（交付物：对比表）
- 输入：要研究的产品方向
- AI 做什么：搜索竞品、提取功能矩阵、标注差异化
- 你做什么：阅读对比表，形成判断，标注"值得深挖的"
- 终点：一张对比表 + 你的批注

### 第二阶段：架构设计（交付物：系统架构图 + 核心决策记录）
- 输入：竞品扫描结论 + 你的产品方向
- AI 做什么：辅助梳理模块关系、提供架构模式参考
- 你做什么：**这是你的核心价值区**——设计系统结构、决定模块边界、做兼容性和扩展性决策
- 终点：一页架构图 + 3-5 条核心决策记录（为什么这样设计，为什么不选另一种方案）

### 第三阶段：需求拆解（交付物：用户故事地图）
- 输入：架构设计产出
- AI 做什么：根据架构生成结构化的用户故事地图，按 P0/P1/P2 分层
- 你做什么：review 分层是否合理，补充缺失场景
- 终点：一页用户故事地图

### 第四阶段：PRD 生成（交付物：PRD 初稿）
- 输入：前三阶段全部产出
- AI 做什么：基于竞品、架构、用户故事，直接生成 PRD 初稿
- 你做什么：review + 修改。不改底层结构，只改内容
- 终点：可提交评审的 PRD

## 为什么这样设计

1. 每个阶段有明确终点 → 满足闭合逻辑环偏好
2. 你的核心价值（架构设计）在第二阶段集中释放
3. PRD 写作被压缩为"review 阶段"而非"创作阶段"
4. AI 负责消耗型的文档写作，你负责增值型的架构决策

# 信息觅食矩阵

> 解决「什么内容去什么渠道」的问题。按内容类型 → 推荐渠道 → 信噪比 → 适合场景。

## 核心原则

小红书的算法推荐适合生活消费决策，不适合建立系统性认知。不同内容类型有最优渠道，混用等于用螺丝刀砍树。

## 渠道矩阵

### 技术深度内容（原理、架构、源码分析）
| 渠道                  | 信噪比 | 适合场景            |
| ------------------- | --- | --------------- |
| Hacker News         | 高   | 每天扫一眼，看什么在讨论    |
| GitHub Trending     | 高   | 每周看一次，发现新工具     |
| Reddit r/LocalLLaMA | 中高  | AI 模型/工具实战讨论    |
| 技术博客（个人/公司）         | 高   | 深度长文，通过搜索引擎被动发现 |
| V2EX                | 中   | 中文技术圈，偏应用层      |

### 产品/商业认知（策略、方法论、趋势）
| 渠道 | 信噪比 | 适合场景 |
|------|--------|---------|
| Stratechery / Ben Thompson | 高 | 科技战略分析（付费） |
| Lenny's Newsletter | 高 | 产品管理实战（付费） |
| A16Z / Sequoia 博客 | 高 | 投资视角的技术趋势 |
| Reddit r/ProductManagement | 中 | PM 同行讨论，偏吐槽 |
| Medium（产品标签） | 中低 | 需筛选，水分多 |

### AI 应用实践（工具、教程、案例）
| 渠道 | 信噪比 | 适合场景 |
|------|--------|---------|
| X/Twitter（精选关注列表） | 高 | 实时动态，一手信息 |
| Product Hunt | 中高 | 发现新产品，看 launch |
| YouTube（精选频道） | 中 | 动手教程，可视化 |
| 少数派 | 中 | 中文工具测评和效率方法论 |
| 小红书 | 低 | 生活类 AI 应用，信息碎片化 |

### 学术/深度研究（论文、基准测试、综述）
| 渠道 | 信噪比 | 适合场景 |
|------|--------|---------|
| arXiv | 高 | 需要搜特定主题论文 |
| Google Scholar | 高 | 引文追踪，看引用关系 |
| Papers With Code | 高 | 论文 + 代码 + 基准 |
| Semantic Scholar | 高 | AI 驱动的文献搜索 |

### 中文社区
| 渠道 | 信噪比 | 适合场景 |
|------|--------|---------|
| 知乎（精选关注） | 中 | 有深度的长回答，需筛选 |
| 少数派 | 高 | 效率工具、数字生活 |
| V2EX | 中 | 技术讨论，偏工程师 |
| 即刻 | 中低 | 碎片化动态，适合发现新东西 |
| 小红书 | 低 | 仅适合生活消费决策 |

## 你的默认信息流

```
每天早上 9:00 Tech Radar 自动推送（GitHub + HN + Reddit + V2EX + PH）
    ↓
觉得有趣 → 点链接深看
觉得相关 → 让我用 Firecrawl 深度抓取
    ↓
值得保留 → 存 Obsidian（我帮你写摘要）
```

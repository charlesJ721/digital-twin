# 认知勤奋与行动回避：一次行为模式剖析

> 创建：2026-06-09 | 来源：与 Hermes 飞书对话
> 关联：AI 认知与实践路径.md

## 核心发现

识别到一个行为模式：

```
表层：「我在努力了解 AI 的底层原理」
实际：「我在避免面对实践的挫败风险」
自察：「我知道自己正在用认知勤奋掩盖行动回避」
```

## 模式结构

1. **触发条件**：面临需要动手实践的任务（如全屋智能方案落地）
2. **替代行为**：转向研究底层技术细节（Attention 机制、模型上下文构造等）
3. **自我合理化**：「我在理解它的原理，这很重要」
4. **情感避让**：回避「全力以赴后失败」的深层恐惧 + 「做出来不满意」的事后挫败感
5. **认知矛盾**：明知自己在回避，但无法自行跳出循环

## 根因

- 纯粹的理解和思考消耗资源最低
- 实践涉及资源精力的爆发式提升
- 面临做不成的挫败感和做出来不满意的事后同感
- 「不行动」是人格特质中的保守策略

## 建立的外部锚点

与 Hermes 约定「敲打规则」：

| 触发信号 | 回应方式 |
|---------|---------|
| 追问离当前任务三跳以外的技术细节 | 标注层次归属，提议先交付再深挖 |
| 「研究」和「动手」间反复横跳 | 提醒信息已够做 MVP，继续研究进入边际收益递减 |
| 要「完整底层原理」而非「够用的理解」 | 明确当前深度已够做决策，剩余细节踩坑时再回来看 |

**不挡求知，不分价值——只在模式出现时做提醒。**
